# LLMBenchmark

A local Large Language Model (LLM) benchmarking framework designed to evaluate model performance, resource utilization, and hardware efficiency on consumer hardware.

---

# Project Overview

LLMBenchmark is a hardware-aware benchmarking system for locally hosted LLMs running through Ollama.

The goal is not simply to measure response speed, but to understand:

* How much RAM a model consumes
* How long a model takes to load
* How many tokens per second it generates
* How efficiently it uses CPU and GPU resources
* How hardware limitations affect performance
* Which model provides the best balance between quality and resource usage

This project is being developed and tested on:

* MSI GF63 Laptop
* Intel Processor
* 16 GB RAM
* NVIDIA RTX 3050 Laptop GPU (4 GB VRAM)
* Windows 11
* Ollama

# Benchmark Metrics

## Model Metrics

### Load Time

Time required to load the model into memory.

Formula:

Load Time = load_duration / 1e9

Unit:

Seconds

---

### Prompt Tokens

Number of tokens in the input prompt.

Source:

prompt_eval_count

---

### Output Tokens

Number of tokens generated by the model.

Source:

eval_count

---

### Prompt TPS

Prompt processing speed.

Formula:

Prompt TPS = Prompt Tokens / Prompt Eval Time

Unit:

Tokens/Second

---

### Generation TPS

Response generation speed.

Formula:

Generation TPS = Output Tokens / Generation Time

Unit:

Tokens/Second

---

### Total Tokens

Formula:

Total Tokens = Prompt Tokens + Output Tokens

---

### Total Runtime

Formula:

Total Duration = total_duration / 1e9

Unit:

Seconds

# Benchmark Types

## Cold Benchmark

Model is unloaded before test.

Example:

ollama stop qwen2.5-coder:7b

Purpose:

Measures:

* Startup Time
* Initial Memory Allocation
* Load Duration

---

## Warm Benchmark

Model remains loaded in memory.

Purpose:

Measures:

* Pure Inference Performance
* Prompt Processing Speed
* Generation Speed

---

# Current Findings

## Qwen2.5-Coder 7B

Hardware:

* RTX 3050 Laptop GPU
* 16 GB RAM

Observed:

* Peak RAM Usage: ~15 GB
* Load Time: ~173 seconds
* Generation Speed: ~2 tokens/sec

Conclusion:

The model is usable but operates near system memory limits.
